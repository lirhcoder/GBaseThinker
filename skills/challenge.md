# /challenge — 让 Thinker 反驳当前结论

当需要检验某个结论是否站得住脚时，激活此 skill。

## 触发场景
- 用户说"你觉得这个对吗？"
- AI 给出了高置信度的结论
- 讨论中出现了未经检验的共识
- 用户主动要求 Thinker 挑战

## 执行流程

### Step 1: 识别目标
明确要挑战的结论是什么。将它表述为一个可证伪的命题。

### Step 2: 假设攻击
对这个结论执行四重攻击：

**攻击 1: 前提审查**
- 这个结论依赖哪些前提？
- 哪些前提是隐含的、未被明说的？
- 如果某个前提不成立，结论还成立吗？

**攻击 2: 反例构造**
- 能否构造一个满足所有前提但结论不成立的反例？
- 能否找到现实中的反例？

**攻击 3: 边界测试**
- 把条件推到极端（极大、极小、极快、极慢）还成立吗？
- 换一个领域/场景还成立吗？

**攻击 4: 利益审查**
- 谁从这个结论中获益？
- 有没有动机偏误？

### Step 3: 生存评估
经过四重攻击后，结论的状态：

| 状态 | 含义 |
|------|------|
| 坚固 | 经受住了所有攻击，可以提高置信度 |
| 有条件 | 在特定条件下成立，需要标注边界 |
| 脆弱 | 有明显漏洞，需要修正 |
| 倒塌 | 核心前提被推翻，结论不成立 |

### Step 4: 输出

```markdown
---
## Thinker 挑战报告

**目标结论**: {conclusion}
**前提审查**: {premises_found} — 隐含前提: {implicit_premises}
**反例**: {counterexample_or_none}
**边界**: {boundary_test_results}
**利益**: {interest_analysis}
**生存评估**: {坚固 | 有条件 | 脆弱 | 倒塌}
**修正建议**: {如果不是"坚固"，建议如何修正}
---
```

### Step 5: 持久化
- 如果结论"坚固" → 考虑是否加入 beliefs.yaml
- 如果结论"倒塌" → 检查是否有需要更新的现有信念
- 如果发现了新的推理模式 → 写入 patterns.yaml
